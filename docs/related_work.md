# Related Work Survey

## LLM Steganography

Steganography in the context of large language models is a relatively new but rapidly developing field. The core idea is to embed secret messages within the text generated by language models, using the inherent randomness in token selection as a communication channel.

**Early Approaches**

Early work on text steganography predates modern language models and focused on linguistic properties such as synonym substitution, syntactic structure modification, and word ordering. These approaches typically operate on already-generated text and modify it to hide information. However, they often suffer from reduced text quality and are vulnerable to statistical analysis.

**LLM-Based Approaches**

With the advent of large language models, researchers have discovered that the probabilistic nature of token selection provides a natural channel for steganography. The model produces a probability distribution over possible next tokens at each step, and this distribution contains entropy that can be exploited to encode information.

One line of work focuses on using the model's sampling process directly. By carefully choosing which token to select from the top candidates based on the secret message bits, information can be embedded without significantly altering the text's appearance. This approach has the advantage of preserving the model's natural language quality while providing a covert communication channel.

Arithmetic coding and similar techniques have been applied to maximize the utilization of the available entropy in these distributions. By treating the secret message as a compressed bitstream and the token probability distribution as the coding model, these methods can achieve near-optimal embedding efficiency.

**Capacity and Trade-offs**

Research has examined the trade-offs between embedding capacity, text quality, and detection resistance. Higher capacity typically requires using more of the available entropy, which may affect the naturalness of the generated text. Conversely, prioritizing text quality often means sacrificing embedding capacity.

The Ghostext approach aims to achieve "almost perfect" steganography by preserving the model's original token probability distribution while maximizing entropy usage through arithmetic coding.

## LLM Watermarking

LLM watermarking is a closely related field that focuses on embedding detectable markers in AI-generated text rather than hiding secret messages. The goals are different—watermarking aims to identify AI-generated content, while steganography aims to hide arbitrary information—but the underlying techniques share many similarities.

**Statistical Watermarking**

The most prominent approach to LLM watermarking is statistical watermarking, which modifies the token selection process to create detectable statistical patterns. A well-known example is the work by Kirchenbauer and colleagues, which introduces a watermark by splitting the vocabulary into "green" and "red" lists and biasing selection toward green tokens.

These watermarks are designed to be statistically detectable without significantly impacting text quality. Detection typically involves analyzing the text for the expected statistical bias in token choices. The strength of the watermark can be adjusted, creating a trade-off between detection accuracy and text quality.

**Resistance to Attacks**

A significant focus of watermarking research has been on resistance to various attacks, including paraphrasing, deletion, insertion, and substitution. Some approaches are more robust than others, with semantic-level watermarks offering better resistance to paraphrasing attacks compared to token-level watermarks.

**Connection to Steganography**

Both watermarking and steganography operate on the same principle: they manipulate the token selection process to embed information. The main difference is their purpose and threat model. Watermarking is designed to be detectable by authorized parties while being difficult to remove. Steganography is designed to be undetectable by unauthorized parties while remaining recoverable by the intended recipient.

The arithmetic coding approach used in Ghostext shares conceptual similarities with some watermarking schemes, particularly in how they utilize the model's probability distribution. However, Ghostext's goal of preserving the original distribution while maximizing entropy utilization distinguishes it from most watermarking approaches, which typically introduce bias for detection purposes.

## Key Papers and Techniques

Several important papers have contributed to these fields:

- "A Watermark for Large Language Models" by Kirchenbauer et al. established the foundation for statistical watermarking of LLM-generated text.
- Work on "Stable Diffusion Watermarking" by various researchers has influenced approaches to image watermarking, with concepts that carry over to text watermarking.
- Research on "Information-Theoretic Limits of Watermarking" has provided theoretical frameworks for understanding the capacity-detection trade-offs.
- Studies on "Robust Watermarks for LLMs" have explored techniques to maintain watermark detectability under editing and paraphrasing.

The Ghostext approach builds on these foundations but with a different emphasis: rather than introducing detectable patterns for content attribution, it aims to embed arbitrary information while maintaining the model's original output distribution, thereby achieving steganographic security against a passive censor.

## Future Directions

Research in both steganography and watermarking continues to evolve. Open questions include improving resistance to detection, increasing embedding capacity, developing cross-model compatibility, and understanding the theoretical limits of what can be achieved in terms of security and efficiency.

Ghostext's contribution lies in demonstrating that near-optimal steganography can be achieved without compromising the model's natural language quality, by carefully preserving the original token probability distribution while efficiently encoding information through arithmetic coding.
